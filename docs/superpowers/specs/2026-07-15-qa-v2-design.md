# Q&A v2 — design (#433, bundles #413 items 3/5/6/8)

Date: 2026-07-15
Issue: #433
Branch: `feat-qa-v2`

## Problem

Rubric baseline (#413 comment): 1.5b answered 0/6 eval questions — it cannot
hold the citation contract, so every live ask degrades to the deterministic
fallback. Users see a streamed draft answer vanish, replaced by a shallow
"The retrieved story is: …" echo that can present an unrelated story as the
answer. 4b passed 2/6 with uncertainty/contested framing as its dominant gap.

## Decisions (settled in brainstorm)

- Q&A switches to 4b via a dedicated `qa_model` setting; narrative + enrichment
  stay on the warm 1.5b. Rejected: risky-question router (1.5b cites 0/6 —
  routing to it is routing to fallback) and a global `brain_model` swap (breaks
  the Pi nightly RAM budget).
- Streamed draft stays visible but is visibly a draft ("drafting — verifying
  sources…"), swapped for the verified final. Rejected: buffering until
  verified (6–14 s blank stare) and post-hoc correction banner (weak signal).

## 1. Model policy (#413 item 8)

- `settings.qa_model: str = "qwen3.5:4b-q4_K_M"`,
  `settings.qa_min_free_mb: int = 3800` (4b q4 ≈ 3.4 GB; the 1200 MB floor was
  sized for 1.5b).
- `/brain/ask`, `/brain/ask/stream`, and the citation-repair call all pass
  `model=settings.qa_model, keep_alive="0"` — 4b is evicted after every ask;
  it never lingers on the Pi.
- `client.generate_text_stream` gains a `keep_alive: str | None = None`
  parameter (falls back to `settings.brain_keep_alive`), mirroring
  `generate_json`.
- Both ask endpoints gate on `qa_min_free_mb` instead of `brain_min_free_mb`.
- README's brain section documents the policy: 1.5b = warm narrative/enrich,
  4b = per-ask Q&A + nightly validator, keep_alive=0.

## 2. Prompt framing (#413 item 6)

`build_qa_prompt` keeps the one-short-string answer contract but replaces the
single "Flag trust" rule with framing rules:

- Disputed or contested coverage must be flagged as such, naming who says what
  when the context shows it.
- A single-source or low-corroboration claim must be described as reported,
  not established.
- Sensor-confirmed claims should say so; sensor-unconfirmed heavy claims must
  be marked unverified.
- When the context cannot answer part of the question, say what is not known
  rather than guessing.

`build_qa_text_prompt` and `build_citation_repair_prompt` inherit the same
rules (text prompt is derived from the JSON prompt; the repair prompt gets the
same framing block).

## 3. Fallback split (#413 item 3)

New constant in `qa.py`:

```python
NO_EVIDENCE_ANSWER = (
    "I don't have enough local evidence to answer that — "
    "closest stories are listed as sources."
)
```

`build_cited_fallback_answer(stories, question=None)`: when `question` is
given and the top story's title+gist shares zero `_question_terms` with it,
return `NO_EVIDENCE_ANSWER` instead of echoing an unrelated story. With no
question or a term match, current behavior stands.

`requires_citation` exempts `NO_EVIDENCE_ANSWER` exactly as it exempts
`REFUSAL_ANSWER`. `qa_rubric.score_answer` treats `NO_EVIDENCE_ANSWER` as a
refusal (same branch as `REFUSAL_ANSWER`) so the eval keeps scoring
consistently.

`_checked_ask_answer` in `api.py` passes the question through to the fallback.

## 4. Draft streaming UX (#413 item 5)

Wire format unchanged (`sources` → `delta`* → `final`). Frontend only:

- While deltas render (before `final`), the answer text is dimmed/italic with
  a "drafting — verifying sources…" tag.
- On `final`, the tag disappears and the verified answer renders normal-style.
- The existing non-stream fallback path renders as today.

## Testing

- `tests/test_brain_qa.py`: prompt contains framing rules; fallback returns
  `NO_EVIDENCE_ANSWER` for a term-mismatched question, keeps story echo for a
  matching one; `requires_citation` exempts the new constant.
- `tests/test_brain_qa_rubric.py`: `NO_EVIDENCE_ANSWER` scores as refusal.
- `tests/test_brain_ask_api.py`: ask endpoints call the client with
  `model=qa_model, keep_alive="0"`; gate uses `qa_min_free_mb`.
- `osint-frontend/lib/brainAsk.test.mts` (+ component behavior): draft state
  until final.
- Post-merge: `make brain-qa-eval` rerun; before/after rubric table to #413.

## Out of scope

Retrieval intent biasing (#413 item 2), sentence-level claim checks (item 4),
coverage-bias context (item 7), freshness ranking + timestamps/links (separate
issues).
