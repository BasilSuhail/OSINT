# Q&A Phase B — question-driven retrieval implementation plan

Goal: make `/brain/ask` retrieve question-relevant recent stories from the local DB
before prompting the 1.5b. Keep Phase A source shape and citation guardrails.

## Constraints

- No new dependencies.
- No live web.
- Keep 1.5b.
- Preserve graceful HTTP 200 failure paths with `sources: []`.
- Cross-dialect: SQLite tests, Postgres production.
- Reuse Phase A trust/source story shape.

## Tasks

1. Add local query parsing and retrieval helpers in `app/brain/qa.py`.
2. Extend `build_qa_stories(..., question=None)` and `build_qa_context(..., question=None)`.
3. Thread `req.question` through `POST /brain/ask`.
4. Add tests for relevant-story selection, member metadata matching, fallback behavior,
   and endpoint question threading.
5. Verify backend tests, ruff, frontend tests/typecheck, live smoke if local model/API
   can be run without triggering macOS fork crash dialogs.

## Acceptance

- A query about a quieter matching story returns that story before a louder unrelated
  story.
- A query with no useful terms keeps the Phase A loudest-story behavior.
- A query with useful terms and no local match returns no stories.
- `/brain/ask` sources are now based on question-retrieved context.
