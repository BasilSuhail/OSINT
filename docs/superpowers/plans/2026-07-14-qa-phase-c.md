# Q&A Phase C — bigger model evaluation plan

Goal: produce a local artifact comparing the 1.5b Q&A brain model with the validator's
4b model on identical retrieved contexts.

## Tasks

1. Add `app/brain/qa_eval.py`.
2. Add tests for latency/citation recording, model/question matrix expansion, and
   Markdown rendering.
3. Add `make brain-qa-eval`.
4. Add design docs.
5. Run backend tests, ruff, frontend checks, and live eval if local Postgres/Ollama are
   available.

## Acceptance

- Running `make brain-qa-eval` writes JSON and Markdown reports.
- The harness passes a model override into the existing local Ollama brain client.
- Candidate calls use `keep_alive="0"` so eval does not leave either model resident.
- The report includes invalid citation counts for the same post-check discipline used
  by `/brain/ask`.
