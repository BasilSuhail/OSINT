# Frontend setup

The dashboard is a Next.js 15 app generated by [v0.dev](https://v0.dev). It reads directly from Supabase via the public anon key — no FastAPI or backend deployment needed for Phase 1.

## Architecture

```
Browser
   │
   ├── @supabase/supabase-js  ─→  Supabase REST  ─→  Postgres
   │                                 (RLS: anon SELECT on events + scores)
   │
   └── react-map-gl + MapLibre GL  → renders world map + markers
```

Two tables exposed publicly via RLS (see `supabase/02_rls_policies.sql`):

- `events` — raw OSINT events (lat / lon / severity / category / payload)
- `scores` — composite stress per (country, month)

Everything else (labels, ingest_health, etc.) is locked down. Service-role queries from the Python backend still work because service_role bypasses RLS.

## Phase 1 — pages

| Page | What it shows | Data |
|---|---|---|
| `/` | World map: country choropleth from latest composite score, point markers for recent hazard events | `scores`, `events` filtered to category=hazard |
| `/articles` | Card list of recent geopolitical events with source URL | `events` filtered to category=geopolitical |

## Setup

1. **Apply RLS policies** in Supabase
   ```sql
   -- Supabase SQL Editor → paste supabase/02_rls_policies.sql → Run
   ```

2. **Get the Supabase public keys**
   - Supabase project → **Project Settings** → **API**
   - Copy **Project URL** (looks like `https://wjschgweqijnzumhxcml.supabase.co`)
   - Copy **anon public** key (long JWT starting with `eyJ...`)

3. **Generate the frontend with v0**
   - Open [v0.dev](https://v0.dev)
   - Paste the contents of [`v0-prompt.md`](v0-prompt.md) verbatim
   - v0 generates the Next.js app
   - Click "Add to project" → creates a new repo

4. **Set frontend env vars**

   In v0 / Vercel / local `.env.local`:

   ```bash
   NEXT_PUBLIC_SUPABASE_URL=https://YOUR_PROJECT_REF.supabase.co
   NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=eyJ...
   ```

5. **Deploy to Vercel** (free)
   - The v0 project deploys with one click
   - Vercel project settings → Environment variables → add the two `NEXT_PUBLIC_*` keys
   - Redeploy

6. **Done**. The dashboard reads live data from your Supabase, refreshes every 60 seconds, shows country shading by composite score, hazard markers, and a recent-events feed.
